# Getting Support

If you need help with **Better Than Claude Skills**, you've come to the right place. We want to ensure you have the best possible experience using and building upon this cognitive architecture.

## 📚 Where to look first

Before reaching out for help, please check if your question has already been answered:

1. **[The README](../README.md)**: Start here for an overview of the project and quick-start instructions.
2. **[The Architecture Guide](../ARCHITECTURE.md)**: For deep technical understanding of how Progressive Disclosure works and how to load skills.
3. **[The Skills Index](../SKILLS_INDEX.md)**: A complete reference for all available skills and their use cases.
4. **[Examples](../EXAMPLES.md)**: Browse our library of high-fidelity prompts and multi-skill workflows.
5. **[Issue Tracker](https://github.com/shivam990q/better-than-claude-skills/issues)**: Search past issues to see if someone else has reported the same problem or asked the same question.

## 🐛 Bug Reports

If you've found a bug (e.g., a skill hallucinating, a broken workflow, or missing context), please open an issue using our [Bug Report Template](https://github.com/shivam990q/better-than-claude-skills/issues/new?template=bug_report.md).

**What makes a good bug report?**
- A clear, descriptive title.
- The exact prompt or context you used.
- The LLM you are using (e.g., Claude 3.5 Sonnet, Claude 3 Opus, GPT-4o).
- Expected behavior vs. Actual behavior.

## 💡 Feature Requests & New Skills

Have an idea for a new skill? Want to improve an existing one? We'd love to hear it!
Please use the [Feature Request Template](https://github.com/shivam990q/better-than-claude-skills/issues/new?template=feature_request.md) to propose your idea. 

*Tip: If you're feeling adventurous, try using the [`skill-creator`](../skill-creator/) meta-skill to build your idea first, and then submit it as a Pull Request!*

## 🔒 Security Vulnerabilities

If you discover a security vulnerability (e.g., a skill prompting malicious code execution, credential leakage, or unsafe behavior), **do not report it in the public issue tracker.** 

Please refer to our [Security Policy](../SECURITY.md) for instructions on how to responsibly disclose the issue to our team.

## 💬 Community & Discussion

Currently, we manage discussions via GitHub Issues. If you have a question that isn't a bug report or a feature request, feel free to open a regular issue and tag it as a `question`.

We are considering launching a Discord server for real-time collaboration. If you are interested, please let us know in the issues!
